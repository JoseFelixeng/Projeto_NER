RESUMO A visualização de sinais em tempo real é um fator crucial quando se deseja entender o comportamento de sistemas, uma vez que, a análise de dados instantâneos permite que o operador detecte padrões e tome decisões de maneira mais rápida e oportuna. Nesse contexto, este trabalho apresenta o desenvolvimento de um sistema supervisório para sinais digitais. A m de atingir esse objetivo, fez-se uso do Processador Digitais de Sinais (DSP) modelo TMSFpara enviar dados via Interface de Comunicação Serial (SCI) para um computador a parte. Em seguida os dados são interpretados e exibidos utilizando-se a biblioteca PyQT. Palavras-chaves: Sinais; DSP; Supervisório; SCI; PyQt. ABSTRACT The visualization of real-time signals is a crucial factor in understanding system behavior, as the analysis of instantaneous data enables operators to detect patterns and make decisions more quickly and eectively. In this context, this work presents the development of a supervisory system for digital signals. To achieve this goal, a Digital Signal Processor (DSP) model TMSFwas utilized to send data via a Serial Communication Interface (SCI) to an external computer. The data is then processed and displayed using the PyQtlibrary. Keywords: Signals; DSP; Supervisory; SCI; PyQt. INTRODUÇÃO O processamento digital de sinais é um ramo da ciŸncia da computação que trata da matemática, dos algoritmos e das técnicas utilizadas para manipular sinais que geralmente foram convertidos para o formato digital. (YU et al., ). Os dispositivos projetados para lidar especicamente com a manipulação de sinais são chamados de Processadores Digitais de Sinal (do inglŸs, Digital Signal Processors - DSP’s) e podem ser aplicados em diversas áreas como: comunicação, medicina, geofísica e engenharia. Durante a execução de um sistema que utiliza o DSP, o processamento de sinais ocorre de forma sequencial e frequentemente em alta velocidade, o que diculta a análise em tempo real do impacto desses sinais no desempenho e no comportamento geral do sistema. Nesse contexto, sistemas supervisórios apresentam-se como ferramentas relevantes ao viabilizar o monitoramento contínuo e eciente dos sinais, sendo uma abordagem que supera não só as limitações das análises oﬄine, mas também contribui para uma maior precisão e conabilidade na tomada de decisões. Dessa forma, este trabalho propõe o desenvolvimento de um sistema supervisório capaz de interpretar sinais digitais provenientes do DSP e exibi-los em grácos, utilizando a comunicação serial do DSP para a transmissão dos dados durante a execução dos programas. .Contextualização Este trabalho foi desenvolvido em parceria com o Laboratório de Avaliação de Medição em Petróleo (LAMP) da Universidade Federal do Rio Grande do Norte - UFRN. A mensagem, disposta como elemento central na Figura , diz respeito a informação que se deseja transmitir e pode possuir diversos formatos, como: textos, números, imagens, áudio e vídeo. O dispositivo responsável por enviá-la é chamado de remetente ou transmissor, ao passo que o dispositivo encarregado de recebŸ-la é nomeado destinatário ou receptor. Interligando transmissor ao receptor, tem-se o meio ou linha de transmissão, sendo este o caminho físico por onde a mensagem se desloca. Alguns exemplos desse componente são o cabo coaxial e o o par trançado. Por m, para que a mensagem enviada seja compreendida corretamente pelo destinatário, é necessário que haja um protocolo na comunicação. Este elemento estabelece um conjunto de regras que os dispositivos devem seguir para se comunicarem de maneira correta. ..Fluxo de dados O uxo de dados refere-se ao movimento de informações entre elementos que compõem a comunicação e pode ocorrer em trŸs modos: simplex, half-duplex e full-duplex. A Figura mostra como são realizadas as conexões em cada um deles. Capítulo . Fundamentação Teórica Figura – Modos do uxo de dados Fonte: Elaborada pelo autor. No modo simplex a comunicação é feita em um único sentido e somente um dos dois dispositivos pode transmitir dados, enquanto o outro apenas recebe. Já no modo half-duplex, cada dispositivo pode transmitir e receber dados, desde que essas tarefas não ocorram ao mesmo tempo, logo, quando um está enviando o outro está recebendo e vice-versa. No modo full-duplex, os dois dispositivos podem enviar e receber dados simultaneamente (FOROUZAN, ). ..Transmissão digital de dados A transmissão digital de dados é o envio de informações em formato de dados digitais, que são representados por sequŸncias de bits ’s e ’s. Ela pode ser realizada em dois modos: paralelo ou serial. Para Forouzan (), a transmissão paralela é um método em que bits são enviados por n canais de comunicação a cada pulso de clock, enquanto na transmissão serial um bit segue o outro, sendo necessário apenas um canal. A Figura ilusta como são feitas as ligações para uma transmissão paralela com canais e para a transmissão serial. Capítulo . Fundamentação Teórica Figura – Conexões físicas nas comunicações Parelela e Serial Fonte: (FOROUZAN, ) Ainda segundo Forouzan (), em termos de usabilidade, as principais diferenças entre os dois modos de transmissão de dados são a velocidade e o custo. A transmissão paralela possui a capacidade de aumentar a velocidade de transferŸncia em um fator de n em relação à transmissão serial. Porém, essa abordagem vem acompanhada de um alto custo de transmissão, uma vez que são necessários mais canais apenas para enviar o uxo de dados. Desse modo, a comunicação paralela geralmente é limitada a curtas distâncias, enquanto a comunicação serial é mais ecaz em longas distâncias. Além de apresentar redução nos curtos e simplicidade, a comunicação serial ocorre de duas maneiras que inuenciam diretamente sua eciŸncia e aplicabilidade, sendo elas: comunicação serial síncrona e assíncrona. Conforme Axelson (), em um protocolo síncrono, a interface possui uma linha de clock controlada por um dos dispositivos e todos os bits transmitidos são sincronizados com esse clock. Assim, a transmissão de cada um deles é tida como válida em um instante especíco após a transição de subida ou descida do sinal da linha. Algumas interfaces seriais síncronas são a IC (Inter-Integrated Circuit), SPI (Serial Peripheral Interface) e Microwire. Já sobre o protocolo assíncrono, Axelson () aponta que não há a presença de Capítulo . Dessa forma, a sincronia entre os dispositivos é feita a partir do envio de um bit especíco chamado bit de início. Dentre os exemplos de comunicação assíncrona existem a UART e RS-. ..Taxas de Transmissão As taxas de transmissão determinam a velocidade com que ocorre a transferŸncia de dados. Desse modo, a literatura aborda dois conceitos importantes envolvendo essas taxas: o bit rate e o baud rate. Segundo Axelson (), bit rate é o número de bits que são transmitidos ou recebidos por unidade de tempo, sendo geralmente expresso em bits por segundo, enquanto o baud rate, diz respeito ao número de transições de sinais por segundo. Neste trabalho, cada transição de dados representa a mudança de um bit, fazendo com que bit rate e baud rate sejam os mesmos. Portanto, o termo adotado pra se referir a taxa de transmissão será baud rate. .UART A UART (Universal Asynchronous Receiver/Transmitter) é uma interface de comu- nicação serial assíncrona que dene um protocolo para a troca de dados entre dispositivos de maneira sequencial, bit a bit. Os elementos que a compõem são um transmissor (TX), um receptor (RX) e um o de terra. A conexão física entre eles é feita com o transmissor de um dispositivo conectado ao receptor do outro (assim como mostrado no formato Full Duplex da subseção ..) e com um aterramento comum. ..Processo de Comunicação Tendo em vista a característica assíncrona da UART, as informações transmitidas seguem um padrão de formatação que as dividem em blocos geralmente chamados de frames. Cada frame é formado por: um bit de início, bits de dados (que podem ser chamados de palavra), um bit opcional de paridade e um ou mais bits de parada. A partir da quantidade de bits conguráveis, existem diversos tipos de formatos para os frames. Em um dos mais comuns, o -N-, o frame possui oito bits de dados, nenhum bit de paridade e um bit de parada. Tal formato tem seus níveis lógicos em uma Capítulo . Para indicar o início de uma transmissão, o transmissor envia um sinal de referŸncia conhecido como bit de início, que possui nível lógico baixo () e é responsável por alertar ao receptor sobre a chegada de um novo frame. A partir da chegada do bit de início, o receptor tenta amostrar os bits seguintes considerando o período do bit, que diz respeito ao tempo que se leva para transmitir um único bit e matematicamente pode ser representado como o inverso do baud rate. Em seguida, são enviados bits de dados em sequŸncia começando pelo bit menos signicativo. Os bits de dados contém a informação útil que se deseja transmitir e sua quantidade pode variar de a , a depender do dispositivo ao qual a UART está vinculada. Após os bits de dados, o transmissor envia o bit de parada para nalizar a trans- missão. Este bit possui valor lógico alto e pode ser de uma, uma e meia ou duas vezes o tempo do período do bit. Além dos bits explicitados, é importante comentar sobre o bit de paridade, que não foi utilizado no exemplo em questão. Tal bit, se congurado, atua como uma forma simples de vericação de erros, sendo calculado pelo transmissor com base na palavra e adicionado ao frame depois desta e antes do bit de parada. Desse modo, existem quatro tipos de paridade: par (even), ímpar (odd), de marca e de espaço, representadas nos formatos de frames respectivamente pelas letras "E", "O", "M" e "S". Em contextos onde a paridade é denida como par ou ímpar, o número total de bits "" no frame tem que ser par para o primeiro caso e ímpar para o segundo. Assim, o bit paridade é congurado como ou para fazer com que o valor total de bits atinja a paridade desejada. Nos casos em que a paridade é denida como de marca, o bit de paridade é sempre e quando é denida como de espaço, o bit de paridade é sempre . Esse tamanho foi escolhido para acomodar até sete dígitos da representação numérica, garantindo um equilíbrio entre o espaço ocupado e a capacidade de representar os números mais comuns com precisão adequada. O caractere adicional é reservado para o terminador nulo (\), utilizado em C para indicar o nal da string. Dito isso, a função numeroParaString() começa vericando se o número é negativo, adicionando o sinal de menos à primeira posição do vetor de char em caso armativo. Em seguida, processa a parte inteira e a parte decimal do número separadamente. Para a parte inteira, os dígitos são extraídos e convertidos para caracteres, sendo armazenados um a um no vetor de char. Caso haja uma parte decimal, ela é processada até quatro dígitos após o ponto decimal, convertendo cada dígito para o formato de caractere. Por m, a função adiciona um caractere nulo (\) ao nal da string para indicar o término. ..Funções para envio de dados O envio de dados no módulo SCI é realizado pela integração entre as funções scia_xmit() e scia_msg(), utilizando o registrador SCITXBUF como ponto central. O SCITXBUF armazena temporariamente os dados a serem transmitidos, transferindo-os para a FIFO de transmissão, que organiza o envio sequencial pelo barramento serial. A função scia_xmit() realiza a transmissão de um único caractere, garantindo que a FIFO de transmissão esteja disponível antes de carregar o dado no SCITXBUF. CONCLUSÃO A implementação do sistema supervisório para monitoramento de sinais provenientes do DSP foi concluída com Ÿxito, uma vez que foi possível utilizar o projeto proposto em um caso prático onde se observou a exibição de curvas que permitem avaliar o comportamento de variáveis escolhida e em diferentes formatos. Contudo, algumas limitações foram identicadas e apontam para possíveis melhorias. A velocidade de transferŸncia de dados pela comunicação SCI utilizada mostrou-se limitada, resultando na perda de informações, pois novos dados eram gerados enquanto informações anteriores ainda estavam sendo transmitidas. Uma solução para esse problema seria empregar drivers que suportem taxas de comunicação mais altas ou adotar protocolos mais rápidos, como SPI ou IC. Além disso, embora a taxa de atualização de um segundo tenha sido adequada para os objetivos iniciais, uma maior uidez na exibição dos dados seria desejável, o que poderia ser alcançado com otimizações no software ou no método de visualização utilizado. Assim, os resultados obtidos demonstram o potencial do sistema, mas eviden- ciam oportunidades para aprimoramentos futuros, visando maior eciŸncia e precisão no monitoramento.
