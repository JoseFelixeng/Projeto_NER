Resumo
Este trabalho apresenta a concepção e implementação de uma API REST conteineri-
zada voltada para servir um modelo de Machine Learning, e a integração dessa API com
práticas de observabilidade e monitoramento utilizando o ElasticAPM e o Elasticsearch.
O serviço de inferência é desenvolvido utilizando o framework FastAPI, escolhido por
sua eﬁciência, facilidade de uso e versatilidade.
A instrumentação da API com o ElasticAPM visa coletar métricas detalhadas de per-
formance, incluindo tempos de resposta, distribuição de spans e erros, permitindo uma
análise aprofundada do comportamento da aplicação em tempo real. Esses dados de mo-
nitoramento são armazenados no Elasticsearch, que oferece uma solução escalável e eﬁ-
ciente para indexação e busca de grandes volumes de dados, facilitando a análise e a
visualização através de ferramenta Kibana e o gerenciamento em ambientes voltados a
microsserviços.
Palavras-chave: Machine Learning, API REST, FastAPI, ElasticAPM, Elasticsearch,
MLops, observabilidade, monitoramento, conteinerização, Docker.
Abstract
This work presents the design and implementation of a containerized REST API de-
signed to serve a Machine Learning model, along with the integration of this API with
observability and monitoring practices using ElasticAPM and Elasticsearch. The infe-
rence service is developed using the FastAPI framework, chosen for its efﬁciency, ease of
use, and versatility.
The instrumentation of the API with ElasticAPM aims to collect detailed performance
metrics, including response times, span distribution, and errors, enabling in-depth analy-
sis of application behavior in real-time. This monitoring data is stored in Elasticsearch,
which offers a scalable and efﬁcient solution for indexing and searching large volumes of
data, facilitating analysis and visualization through the Kibana tool and management in
microservice-oriented environments.
Keywords: Machine Learning, API REST, FastAPI, ElasticAPM, Elasticsearch, MLops,
observability, monitoring, containers, Docker.

2.1
Contêineres Docker
Docker é uma plataforma de virtualização ao nível de sistema operacional, baseada
em contêineres, que permite o desenvolvimento, o teste e a implantação de aplicações
de forma consistente em diferentes ambientes. Utiliza a tecnologia de contêineres do
Linux para empacotar o software em um ambiente completo, que inclui todos os seus
componentes e dependências, garantindo que a aplicação funcione da mesma maneira
independentemente do ambiente em que é executada.
Para aplicações de Machine Learning, o Docker é particularmente útil, pois facilita a
reprodução de ambientes de desenvolvimento em ambientes de produção, minimiza con-
ﬂitos de dependência e permite a rápida escalabilidade de serviços. Além disso, o Docker
é um pilar para a implementação de pipelines de CI/CD, automatizando a implantação e
os testes de aplicações.
2.2
Elasticsearch, Kibana e APM server
No contexto de observabilidade e gestão de performance de aplicações, como é o
caso da inference-api, o Elasticsearch desempenha um papel fundamental ao servir como
repositório central para logs e dados de APM (Application Performance Management). A
ferramenta de APM, ElasticAPM, coleta informações detalhadas sobre o desempenho da
aplicação, incluindo métricas de transações, erros e estatísticas de utilização de recursos,
e armazena esses dados no Elasticsearch.
A importância do Elasticsearch nesse cenário reside em sua habilidade de lidar com
a ingestão e o armazenamento de uma grande quantidade de dados gerados continua-
mente pela instrumentação da API. Esses dados incluem logs de transações, métricas
de performance e qualquer incidente de erro que ocorra durante a operação de APIs
REST. A capacidade de armazenar e indexar esses dados de forma estruturada permite
que sejam realizadas buscas complexas e análises em tempo real, o que é essencial para
a detecção de anomalias de performance e para a tomada de decisão baseada em dados
concretos.(Kathare et al. 2020)
Além disso, o Elasticsearch é parte da Elastic Stack, que inclui o Kibana, uma inter-
face de usuário que permite a visualização e análise interativa dos dados armazenados no
Elasticsearch (Akdal et al. 2018). Com o Kibana, é possível criar dashboards personali-
zados que exibem métricas relevantes para o monitoramento da APIs REST, facilitando a
identiﬁcação de padrões e tendências que podem indicar a necessidade de otimizações.
De acordo com o artigo "A Basic Introduction to DevOps Tools", o autor conclui
que um stack formado por Elasticsearch e Kibana pode ser uma boa alternativa (Jagadish
2015), tornando essa a ferramenta mais versátil e o melhor candidato para os objetivos na
questão de monitoramento.
Outros casos de de uso do ElasticStack para monitoramento é descrito em "Monitoring
of IaaS and scientiﬁc applications on the Cloud using the Elasticsearch ecosystem"(Bagnasco
et al. 2015), no qual dados e uma Cloud privada é ingerido no ElasticSearch
Alem disso, outro caso de sucesso utilizou o cliente ElasticAPM, para melhorar a
observabilidade em sistemas SCADA complexos, em que a quantidade de dados crescia
exponencialmente. No paper, os autores concluem que a abordagem de integrar a ferra-
menta ofereceu modularidade e insights imediatos. (Leitner et al. 2012)
Em resumo, o Elasticsearch é uma peça central na infraestrutura de monitoramento de
aplicações de Machine Learning, oferecendo uma solução robusta para armazenamento,
busca e análise de logs e dados de performance. Sua integração com ferramentas de APM
como o ElasticAPM permite um monitoramento abrangente e aprofundado, essencial para
manter a saúde e a performance de sistemas críticos.

FastAPI

No contexto de uma "API de inferência"genérica para Machine Learning, a utilização
de um framework RESTful como o FastAPI oferece vantagens signiﬁcativas. FastAPI é
conhecido por sua eﬁciência e por sua capacidade de tornar o desenvolvimento de APIs
com Python mais ágil e menos propenso a erros, graças à sua integração com sistemas de
tipagem do Python e ferramentas de validação de dados. Além disso, ela pode ser até 45%
mais performática em modelos de ML do que se usado frameworks como Flask(Bansal &
Ouda 2022), o que faz o framework ideal para a construção de uma API REST.
Um serviço de inferência desenvolvida com FastAPI pode expor endpoints que permi-
tem a interação com modelos de ML, aceitando dados de entrada e retornando predições
ou análises. A auto-documentação gerada pelo FastAPI, através de interfaces como o
Swagger UI, facilita o entendimento e a utilização da API por terceiros, promovendo a
interoperabilidade e a reutilização de código.
A natureza stateless das APIs REST, como a que seria implementada para a "API de
inferência", é ideal para ambientes de nuvem e escalabilidade horizontal, onde múltiplas
instâncias de um serviço podem ser distribuídas para atender a demandas variáveis de
solicitações. Cada instância pode processar requisições independentemente, sem a ne-
cessidade de compartilhar estado entre si, o que simpliﬁca o design e a manutenção do
sistema.
Quando integrada com ferramentas de APM como o ElasticAPM, uma "API de infe-
rência"baseada em FastAPI pode ser monitorada em tempo real, permitindo a coleta de
métricas críticas, como o tempo de resposta de cada endpoint, a utilização de recursos do
servidor e a identiﬁcação de possíveis pontos de falha ou lentidão. Esses dados são então
armazenados e indexados pelo Elasticsearch, possibilitando consultas e análises detalha-
das que auxiliam na otimização da performance da API.
Em resumo, a utilização de uma API RESTful com FastAPI para o desenvolvimento
de um serviço de inferência baseado de Machine Learning proporciona uma solução ro-
busta e escalável, ao mesmo tempo em que facilita a implementação de práticas de mo-
nitoramento e observabilidade essenciais para a Engenharia de Software e DevOps. Isso
assegura que a "API de inferência"não apenas ofereça uma interface acessível e bem do-
cumentada para os modelos de ML, mas também mantenha um alto nível de performance
e conﬁabilidade em produção.

Integração com ElasticAPM
A implementação do monitoramento de performance da API REST de machine lear-

mações do código instrumentalizado do container e armazená-las no ElastiSearch, desse

funcionamento do ElasticSearch, tais como criação de roles extras, credenciais, índices
extras ou política de retenção. Para o propósito deste trabalho as conﬁgurações padrões e
mais básicas foi utilizada.1

4.2
Visualização no Kibana
No Kibana, criamos visualizações para exibir o tempo de resposta médio, a distribui-
ção dos tempos de processamento do span process_prediction, e a frequência das requisi-
ções ao longo do tempo. Essas visualizações ajudaram a identiﬁcar rapidamente qualquer
anomalia no comportamento da API, como latências inesperadas ou erros de processa-
mento.
A ﬁgura 4.1 ilustra a latência total de toda a requisição (barra verde), bem como
a latência de um trecho de código especiﬁco (barra vermelha), desse modo é possível
identiﬁcar facilmente onde os principais gargalos se encontram.
Além disso, o tipo de requisição (POST), o endpoint atingido (/predict) e o cliente
utilizado (Postman) são capturados e aparecem na interface, permitindo fazer analises de
correlação entre as variáveis.
Uma principais vantagens do uso do ElasticAPM em conjunto com o Kibana é a ca-
pacidade de oferecer uma visualização uniﬁcada de múltiplos serviços dentro de uma
arquitetura de microserviços. No contexto dos experimentos realizados, além da nossa
aplicação FastAPI denominada inference-api, também é possível monitorar e visualizar
dois serviços adicionais em Java, demonstrando a ﬂexibilidade e a amplitude da solução
de monitoramento. Os serviços que aparecem na imagem são apenas ilustrações da capa-
cidade da aplicação, não sendo serviços realmente funcionais ou relevantes no trabalho.
