A InteligŸncia Articial (IA) está transformando diversos setores, remodelando diariamente
o que entendemos por “tecnologia”. Sua aplicação na análise da linguagem natural é de
suma importância, com essa área se desenvolvendo exponencialmente todos os dias. Neste
contexto de desenvolvimento„ este trabalho tem por objetivo analisar e comparar diferentes
abordagens para o Processamento de Linguagem Natural (PLN) e classicação de texto
aplicadas às notícias encontradas no site da Universidade Federal do Rio Grande do Norte
(UFRN), classicando-as entre os temas de “ciŸncias”, “eventos”, “vagas” e “informes”,
além de disponibilizar interfaces com os melhores modelos. A metodologia adotada abrange
desde a escolha das tecnologias até a implementação das interfaces. Nesse sentido, o Data
Version Control (DVC) foi utilizado para o armazenamento e versionamento dos dados no
desenvolvimento dos modelos, o GitHub para o versionamento de código, o Weight and
Biases para versionamento e obtenção de grácos relacionados ao treinamento dos modelos,
além dos dados utilizados em conjunto com as interfaces, a API da OpenAI para realizar
os embeddings dos textos, e o Gradio, em conjunto com o Hugging Face, para construir
e disponibilizar os modelos. Por m, os resultados obtidos demonstram que a maioria
dos modelos apresentam uma alta acurácia, entretanto, aquele que utiliza o algoritmo de
XGBoost apresentou um melhor resultado, conseguindo classicar, com precisão, os textos
em linguagem natural. Visto que a IA está em constante crescimento e surgem cada vez
mais abordagens para um certo m, a discussão sobre qual é a melhor, a partir de um

Articial Intelligence (AI) is transforming various sectors, reshaping what we understand
as “technology” on a daily basis. Its application in natural language analysis is of utmost
importance, with this eld evolving exponentially every day. In this developmental context,
this work aims to analyze and compare dierent approaches to Natural Language Processing
(NLP) and text classication applied to the news found on the website of the Federal
University of Rio Grande do Norte (UFRN). The classication is performed among the
themes of “sciences”, “events”, “job openings” and “announcements”, while also providing
interfaces with the best models. The adopted methodology encompasses everything from
the choice of technologies to the implementation of interfaces. In this sense, Data Version
Control (DVC) was used for data storage and versioning during the development of the
models, GitHub for code versioning, Weight and Biases for versioning and generating
graphs related to model training, the data used alongside the interfaces, OpenAI’s API for
embedding the texts, and Gradio, in conjunction with Hugging Face, to build and deploy
the models. Finally, the results show that most models achieved high accuracy; however,
the one utilizing the XGBoost algorithm delivered the best performance, successfully
classifying natural language texts with precision. Considering that AI is continuously
growing and new approaches are constantly emerging, the discussion on which is the best
approach for a given context is indispensable.

Neste capítulo, busca-se contextualizar a crescente presença da InteligŸncia Articial
(IA) e suas implicações no cotidiano da sociedade contemporânea. Em particular, desca-se
o uso da IA no Processamento de Linguage Natural (PLN). Além disso, apresenta-se a
motivação deste tabalho, os objetivos, metodologia e a estrutura desta pesquisa.
1.1
Contextualização
A IA é o domínio da CiŸncia e Engenharia preocupado com o desenvolvimento de
sistemas que permitem que computadores simulem a capacidade de pensamento lógico
e inteligŸncia humana (TECUCI, 2012). Nesse contexto, a classicação textual é de
importância fundamental, e vem se destacando graças aos desenvolvimentos nos campos
de mineração de texto e PLN. Os métodos de classicação possuem o objetivo comum de
designar um certo rótulo para um determinado dado de entrada (GASPARETTO et al.,
2022).
A IA passou por avanços signicativos nos últimos anos, na pesquisa teórica e
também nas aplicações práticas. Isso se dá pelo aumento do poder computacional, pelo
grande volume de dados disponíveis e pelas melhorias na metodologia. Combinados, esses
fatores tŸm viabilizado soluções robustas em várias áreas (SAMEK; MÜLLER, 2019).
Consoante Gasparetto et al. (2022), com a grande quantidade de dados textuais
sendo criadas continuamente, há muito tempo superou as soluções manuais para classica-
ção, isso signica que a classicação textual associada à IA não é apenas conveniente, mas
também estritamente necessária.
1.2
Objetivos
1.2.1
Geral
Este trabalho tem por objetivo geral treinar, avaliar e comparar trŸs diferentes
modelos de classicação de texto com base nas notícias da Universidade Federal do Rio
Grande do Norte (UFRN), além de disponibilizar interfaces interativas para utilizá-los.
Esses modelos irão classicar textos entre os temas de “eventos”, “ciŸncias”, “informes” e
“vagas”.
Para alcançar o prósito citado anteriormente, este trabalho se orienta pelos objetivos
especícos a seguir:
• Classicar um número razoável de notícias entre os temas propostos para servir como
base para desenvolver um dos modelos citados, e para calcular a acurácia de cada
modelo treinado.
• Aplicar técnicas de processamento de linguagem natural e de classicação de texto
para desenvolver os objetos de estudo citados.
• Avaliar os modelos de forma isolada, expondo métricas relevantes para análise.
• Comparar os modelos e observar qual apresentou o melhor resultado.
• Disponibilizar trŸs ambientes interativos, utilizando técnicas de Machine Learning
Operations (MLOps), para que o usuário possa classicar textos.
1.3
Classiﬁcação da Pesquisa
Para alcançar os objetivos deste trabalho, foi necessário denir a classicação quanto
ao método de pesquisa. Nesse contexto, no que diz respeito à natureza da pesquisa, ela
se enquadra como aplicada, uma vez que busca produzir conhecimentos aplicáveis para
solucionar um problema especíco: o processamento e classcação de textos (WAZLAWICK,
2009) e também como comparativa, já que visa analisar e interpretar os paralelos entre os

Em relação à abordagem dos dados, a pesquisa foi categorizada como quantitativa,
tendo-se em vista as métricas utilizadas para medição da eciŸncia.

Existe um volume abundante de texto em linguagem natural no mundo conectado,
embora intrínseco a esse volume de dados haja grande conhecimento a ser explorado, está
se tornando cada vez mais difícil de disseminá-lo, especialmente dado um tempo de vida
humana. O PLN visa fazer esse trabalho com eciŸncia e precisão, como um humano faz

Tendo-se em vista essa grande quantidade de documentos textuais diários que é

2020), esses foram os algoritmos escolhidos para as tarefas de classicação das notícias da

Portanto, o trabalho visa observar a eciŸncia e precisão dos modelos desenvolvidos

2.1
Processamento de Linguagem Natural
Segundo Brownlee (2017), o estudo sobre Processamento de Linguagem Natural,
com sigla PLN, tem origem há mais de 50 anos, e é denida como sendo a manipulação
automática da linguagem natural, como fala e texto, por um software. Suas raízes são
provenientes da relação da liguística tradicional com a evolução computacional. Desse
modo, o principal objetivo do PLN é implementar sistemas capazes de interagir com a
linguagem humana, seja ela por texto ou por voz.
A linguística computacional é o estudo da língua usando as ferramentas da ciŸncia
da computação. A linguística clássica pode ser a linguística computacional de hoje, já que
abrange métodos tradicionais quanto abordagens modernas (BROWNLEE, 2017).
Logo, o PLN consiste na manipulação computacional na linguagem natural para
compreender declarações humandas com o intuito de dar respostas úteis, conforme Bird,
Klein e Loper (2009).
2.1.1
Tokenização
A tokenização é uma das muitas etapas possíveis de pré-processamento que podem
ser aplicadas aos textos que vão interagir com um modelo de machine learning. Para Gre-
fenstette (1999), a tokenização signica dividir o texto de entrada, que para o computador
é apenas uma longa string, em subunidades, chamadas tokens.
2.1.2
Lematização
A lematização, segundo Plisson et al. (2004), é o processo de encontrar a forma
normalizada de uma palavra. É um importante passo para o PLN e muitos outros campos
que lidam com a linguística no geral. Essa abordagem também permite gerar palavras
chaves para ferramentas de busca, entre outras aplicações. Consiste em retirar ou adicionar
um suxo a uma palavra para obter sua forma canônica, recorrendo ao innitivo dos
verbos e ao singular dos substantivos e adjetivos.

Embeddings

Com a publicação do artigo Word2Vec pelo Google, houve um marco na utilização
dos embeddings para gerar representações numéricas dos dados. Todavia, foi com a arqui-

Com isso, um embedding é, segundo Boykis (2023), um vetor multidimensional que

diversas tarefas, desde sistemas de recomendação até classicação textual.
2.1.4
Matriz TF-IDF
Em grandes conjuntos de textos, algumas palavras estarão muito presentes, como
“a”, “o”, “e”, “é”, entre outras. Elas carregam, assim, pouca informação sobre o real
conteúdo do documento. Com a pura contagem das palavras, os termos citados que
aparecem em muitos documentos iriam ofuscar aqueles que são raros, mas carregam um
alto valor semântico para o texto (DEVELOPERS, 2024).
TF-IDF é a sigla para Term Frequency - Inverse Data Frequency. A frequŸncia
do termo (TF) representa a frequŸncia com que uma palavra (termo) aparece em um
documento em relação ao número total de palavras do documento que está sendo analisado.
A Equação 2.1 mostra como calcular a TF de uma palavra (TRIPATHI, 2018).
TFi,j = fi,j
(2.1)
Onde:
• fi,j é o número de vezes que a palavra ti aparece no documento dj.
• Nj é o número total de termos no documento dj.
Segundo Tripathi (2018), o inverso da frequŸncia dos dados (IDF) mede a impor-
tância de uma palavra em todo o conjunto de documentos. O objetivo por trás do IDF
é dar uma maior importância para os termos mais raros em detrimento dos comuns. Na
Equação 2.2 é mostrada a fórmula para calculá-lo.
• N é o número total de documentos.
• dfi é o número de documentos em que o termo ti aparece.
Desse modo, para calcular o TF-IDF da palabra i, que pertece ao documento j,
multiplica-se a Equação 2.1 pela Equação 2.2, como mostra a Equação 2.3
TF-IDFi,j = TFi,j × IDFi
(2.3)
O resultado é uma matriz esparsa, ou seja, que possui várias posições iguais a
zero, uma vez que existem muitos termos que não estarão em um determinado documento
textual, o que gera um TF-IDF = 0.
2.1.5
Similaridade de cosseno
A similaridade de cosseno é um componente básico para várias aplicações de mine-

2.1.8
Redução de dimensionalidade: Principal Component Analysis (PCA)
A redução de dimensionalidade pode ser feita de duas formas diferentes: apenas
mantendo as variáveis mais relevantes, sem alterá-las, do conjunto de dados original (essa
técnica é chamada de feature selection), ou reduzindo o número de variáveis no conjunto de
dados transformando as variáveis originais em um novo conjunto, que é uma combinação
dos dados originais (essa técnica é chamada de dimensionality reduction). Neste trabalho
é utilizada a segunda técnica (SORZANO; VARGAS; MONTANO, 2014).
Segundo Sorzano, Vargas e Montano (2014), a redução de dimensionalidade não é
nova na estatística. Na verdade, o Principal Component Analysis (PCA) data de 1901,
desenvolvido por Karl Pearson. Entretanto, conforme cita Kurita (2019), o PCA é uma
ferramenta padrão na análise de dados moderna e é usada por quase todas as disciplinas
cientícas. A ideia central é encontrar um novo sistema de coordenadas que os dados
possam ser representados com muito menos variáveis que possuem um erro signicante
(SORZANO; VARGAS; MONTANO, 2014).
O Objetivo do PCA é identicar a base, ou sistema de coordenadas, mais signicativa
para expressar, de maneira nova, um conjunto de dados. Espera-se que essa nova base
revele a estrutura oculta do conjunto de dados e ltre o ruído (KURITA, 2019).

3.1.1
K-Means
Nesta abordagem de aprendizado não supervisionado, são realizados os passos
mostrados na Figura 4:
1. Mesclar informações: A primeira etapa consiste em unir as informações de título
e corpo da notícia, uma vez que ambas as características são importantes para
identicar o seu tema. Nesse sentido, o título é antecedido de “Título:”, enquanto o
corpo da requisição é antecedido de “Conteúdo”.
2. Remover HyperText Markup Language (HTML): A API responsável por
prover as notícias retorna o texto com tags HTML e com informações desnecessárias,
como fotos e suas fontes. Esse conteúdo seria extremamente prejudicial para o modelo,
meio dos modelos de embeddings da OpenAI. Esses modelos aceitam uma quantidade
máxima de tokens, a depender do modelo utilizado, neste trabalho será utilizado o
modelo “text-embedding-3-large”, que permite uma entrada com no máximo 8191
tokens. Com isso, o texto é convertido em uma representação por tokens e é removido
caso ultrapasse o limite permitido. Para o conjunto de dados que está sendo usado,
não houve remoção.
4. Utilizar os modelos de embeddings da OpenAI: Neste passo, é utilizado, de
fato, o modelo de embedding “text-embedding-3-large”, por meio de requisições para
a API da OpenAI contendo os textos a serem processados. O resultado é versionado
pelo DVC.
5. Transformar a representação vetorial do texto em uma matriz numérica:
Os vetores obtidos na etapa anterior estão armazenados como strings, é necessário
transformá-los em uma matriz.
6. Normalizar dados: Esse é um passo muito importante no preprocessamento dos
dados para a utilização do modelo em questão, uma vez que ele utiliza a distância
Euclidiana para agrupar os dados. Está sendo feita a normalização Z-score, que
consiste em transformar os dados para que contenham média 0 e desvio padrão 1.
O artefato responsável por normalizar a matriz contendo os embeddings é salvo no
W&B para futura utilização.
7. Treinar modelo: O treinamento do modelo K-Means consiste em formar clusters,
ou seja, grupos de dados agrupados de acordo com sua similaridade. Neste caso, a
quantidade de clusters é a mesma quantidade de temas possíveis para as notícias:
quatro. Após ajustar o modelo aos dados, é feita a previsão de agrupamento das
notícias, como existem quatro possibilidades, são rotuladas de zero a trŸs. Então,
com esse procedimento concluído, é preciso saber qual tema cada rótulo representa.
Para fazer essa tarefa, são escolhidas cinco notícias, de forma aleatória, de cada
cluster, em seguida, essas notícias são formatadas como na subseção 3.1.1, item (1).
A partir disso, elas são enviadas para o modelo de linguagem GPT-4 da OpenAI,
por meio de sua API, e lhe é perguntado qual o tema das notícias, considerando
as quatro possibilidades. Em função disso, é possível saber o tema que cada cluster
representa. Em seguida, os dados classicados são armazenados no Google Drive por
meio do DVC, assim como no W&B com o intuito de utilizá-los no ambiente de
produção. O modelo também é salvo no W&B.

A partir da matriz de confusão, é possível calcular várias métricas importantes para
medir o quão acertivo o modelo é. Neste trabalho são avaliadas a acurácia, a precisão, a
sensibilidade, ou recall, e o F1-score.
A acurácia mede a proporção de previsões corretas em relação ao total de previsões.
A precisão refere-se à proporção de verdadeiros positivos em relação às previsões positivas,
ou seja, é a relação entre a quantidade de previsões corretas de uma amostra pertencer
à classe observada em relação a quantidade total de intâncias classicadas como sendo
dessa classe. O recall mede a proporção de verdadeiros positivos em relação ao total de
amostras positivas reais, que por sua vez, dizem respeito aos exemplos que são realmente
de uma determinada classe observada. O F1-Score é a média harmônica entre a precisão e
Uma alta precisão indica que poucas notícias de classes diferentes da que se está
sendo observada (classe positiva) foram classicadas como pertencentes a ela (falso positivo).
Por outro lado, o recall, ou sensibilidade, mede o quão bem notícias da classe positiva
estão sendo classicadas como sendo de fato desta classe. As classicações incorretas nesse
contexto são chamadas de falso negativo. Já o F1-score é a média harmônica entre a
precisão e o recall, o que proporciona um equilíbrio entre essas duas métricas e evita que o
foco excessivo em uma delas comprometa a outra.

tendo-se em vista que no texto existem palavras e expressões que evidenciam o tema da
É importante analisar de forma isolada alguns pontos, como a baixa precisão da
classe “ciŸncias” e o baixo recall da classe “informes”. A razão disso ca evidente ao se
analisar a matriz de confusão na Figura 16. Notícias que são do tema “informes” por vezes
são erroniamente classicadas como textos que discorrem sobre ciŸncias, consequentemente,

Isso ocorre por esses textos não possuírem palavras que façam sua clara distinção.
O F1-score mostra um desempenho fraco do modelo ao classicar notícias do tema
“informes”, prejudicado pelo baixo recall. Ele denota, também, um desempenho mediado
para “ciŸncias”, que é prejudicado pela baixa precisão. Enquanto os temas “eventos” e
“vagas” possuem uma boa e ótima média harmônica, respectivamente.

Quando um modelo utiliza o PCA para redução de dimensionalidade, é possível
medir a quantidade de informação perdida nesse processo. Cada componente principal
captura uma fração da variabilidade presente nos dados originais. O PCA seleciona os
componentes que retŸm a maior parte da variância dos dados, descartando os menos
relevantes e, com eles, parte da informação.
O PCA fornece a variância explicada por cada componente principal. Ao somar a
variância capturada por todos os componentes selecionados, obtém-se a variância explicada.
O complemento desse valor corresponde à variância perdida. No caso analisado, os valores
para essas métricas foram 7,87% e 92,13%, respectivamente.
Dessa forma, observa-se uma grande perda de informação ao reduzir os embeddings
para duas dimensões. Esse fato impacta negativamente nas métricas do modelo, como
evidenciado nesta seção.

4.1.5
Comparação dos Modelos
Conforme exibido, o modelo com o algoritmo XGBoost, de forma geral, se mostrou o
mais eciente dentre os trŸs desenvolvidos neste trabalho. O alto desempenho do algoritmo
XGBoost, dado o seu comitŸ de máquinas, contribuiu para este resultado. Também teve
um papel importante o preprocessamento dos dados textuais que diferiu daqueles que
utilizaram o algoritmo K-Means, que zeram uso da API da OpenAI para gerar os
embeddings dos dados para uso no modelo, enquanto o modelo com XGBoost utilizou uma
matriz esparsa contendo a frequŸncia de palavras nos documentos. Os clusters formados
em ambos os modelos com K-Means não foram tão precisos, é o que demonstram os
resultados obtidos. Isso se deve pela quantidade de dimensões dos embeddings. No modelo
sem redução de dimensionalidade, permaneceram muitos ruídos na representação numérica
do texto. Enquanto aquele que faz uso dessa técnica, reduziu muito a dimensão, perdendo
componentes que retiam a maior parte da variância dos dados.

Interfaces do Usuário
As interfaces que permitem o usuário interagir com os modelos usam similaridade
por cosseno para vericar a semelhança entre o texto de entrada do usuário e os textos
armazenados na base de dados. Para tal, foi escolhido um limiar de 85%, ou seja, apenas
textos com uma taxa de semelhança menor que essa são armazenados no W&B para um
posterior novo treinamento.
Ademais, para armazenar esses textos estava sendo utilizado o DVC, todavia, ele
se mostrou lento em produção, ao menos em conjunto com o Google Drive. E para fazer
autenticação de forma automática em produção nessa plataforma, é necessário ter uma
conta de serviço no Google Cloud Plataform (GCP) e anexar ao código um arquivo com
informações privadas. Para o Hugging Face - plataforma utiliza para hospedagem das
interfaces - isso é um problema, uma vez que os usuários podem acessar o código fonte,
logo veriam informações condenciais. O W&B se mostrou mais performático ao salvar
grandes conjuntos de dados, como é o caso daqueles que envolvem o modelo K-Means, já
que os embeddings dos textos são persistidos. Além disso, é mais seguro para o uso neste
trabalho, já que necessita apenas de uma chave de acesso, que é facilmente congurada
como variável de ambiente privada.

A classicação das notícas da Universidade Federal do Rio Grande do Norte (UFRN)
entre “ciŸncias”, “eventos”, “vagas” e “informes”, com K-Means, K-Means com redução de
dimensionalidade e XGBoost como modelos propostos, mostrou resultados promissores,
principalmente com o modelo que utiliza o algoritmo XGBoost para previsão, comprovando,
assim, sua utilidade prátiva na classicação das notícias da universidade.
Ademais, a análise de custo-benefício, considerando o custo em tokens, dado os
novos modelos de embeddings da OpenAI, se mostrou economicamente viável, uma vez que
para processar todas as notícias produzidas até hoje foram gastos U$ 1.448. Esse custo
é necessário para os modelos que usam o algoritmo K-Means, uma vez que apenas eles
fazem uso da API da OpenAI. Além disso, para escalar as soluções, é precido aumentar o
poder de processamento do Weigths & Biases e do Hugging Face aderindo a planos pagos.
Vale ressaltar que as trŸs soluções empregaram ecientemente modelos para a
classicação de textos no que diz respeito às notícias da UFRN. Elas foram contempladas
por uma interface amigável e de fácil utilização criada por meio do Gradio, para interação
com os modelos, e hospedada no Hugging Face.
O modelo que apresentou melhor desempenho, no geral, conforme a acurácia,
precisão, recall e F1-score, foi aquele que utiliza o algoritmo XGBoost. Contudo, apesar dos
outros modelos terem apresentado uma eciŸncia razoável, poderiam ter seus desempenhos
melhorados caso o número de features para a redução de dimensionalidade fosse um valor
intermediário entre dois, que foi o escolhido, e o valor total retornado pela API da OpenAI,
que contém muitos ruídos. Isso é evidenciado com a variância perdida com nesse processo.
Como trabalho futuro, pretende-se treinar um modelo com uma quantidade inter-
mediária de componentes para a redução de dimensionalidade com o intuito de avaliar
se isso resulta em um melhor desempenho. Além disso, o classicador de notícias será
implantado na UFRN para que a comunidade acadŸmica possa utilizá-lo.
