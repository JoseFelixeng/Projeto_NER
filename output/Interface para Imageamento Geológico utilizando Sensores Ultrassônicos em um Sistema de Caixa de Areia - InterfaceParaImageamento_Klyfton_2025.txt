A modelagem física sísmica é uma área da geofísica responsável por modelar como se
comporta o subsolo de determinados locais e uma das formas de se realizar essa modelagem
envolve a utilização de sensores ultrassônicos em um sistema de caixa de areia. Uma
utilização desse sistema de caixa de areia é em batimetria, que é o imageamento geológico
da superfície daquele sistema. O objetivo deste projeto é criar uma interface capaz de
receber os dados de um sistema com sensores ultrassônicos e realizar esta modelagem em
formato de gráco, para que seja possível visualizar de forma completa o imageamento
geológico. Para isso, foi utilizado um protótipo do sistema de medição de caixa de areia,
com um ESP32 e um sensor ultrassônico HC-SR04, e criada uma interface para modelagem
2-D e 3-D utilizando-se o Qt Designer e o VSCode, onde os dados são plotados na interface
de aquisição e podem ser salvos em formatos de texto .csv e de imagem, para estudos
posteriores.
Palavras-chaves: Imageamento Sísmico; Sensores Ultrassônicos; Interface de Aquisição;
Sistema de Caixa de Areia.
Seismic physical modeling is an area of geophysics responsible for modeling how the
subsurface behaves in specic locations, and one of the ways to perform this modeling
involves the use of ultrasonic sensors in a sandbox system. One application of this sandbox
system is bathymetry, which is the geological imaging of the surface of that system. The
goal of this project is to create an interface capable of receiving data from a system
with ultrasonic sensors and performing this modeling in the form of a graph, so that the
geological imaging can be fully visualized. To achieve this, a prototype of the sandbox
measurement system was used, with an ESP32 and an HC-SR04 ultrasonic sensor, and an
interface for 2-D and 3-D modeling was created using Qt Designer and VSCode, where
the data is plotted in the acquisition interface and can be saved in text .csv and image
formats for later studies.
Keywords: Seismic Imaging; Ultrasonic Sensors; Acquisition Interface; Sandbox System.

O sistema de caixa de areia é um modelo de imaginação sísmica de fundamental
importância quando se planeja fazer estudos, ou experimentos, geológicos e geofísicos. Eles
permitem a observação da evolução das estruturas, tornando possível uma interpretação
dos sistemas estruturais mais eciente (KEAREY; AL., 2002). É através deste sistema
que são possíveis simulações de processos geológicos complexos, como formações de falhas,
dobras e zonas de cisalhamento, em ambientes controlados, onde pode-se estudá-los em uma
escala de tempo menor do que a que ocorre na vida real. Por meio deste tipo de sistema é
que são validados os modelos teóricos, comparando dados obtidos em laboratório com os
obtidos em campo, ajustando e melhorando a partir disso as técnicas de interpretação.
Nesse tipo de sistema, quem controla as condições é o construtor do modelo, podendo
comparar os resultados da modelagem com exemplos geológicos naturais.
Além disso, é um equipamento perfeito se tratando de educação e treinamento,
sendo útil para ensinar estudantes e pesquisadores na visualização e entendimento das
forças tectônicas atuantes na formação de estruturas geológicas. Por ser amplamente
utilizado para desenvolver e testar novos métodos sísmicos e equipamentos em menores
escalas e em ambientes mais controlados, antes de serem levados a campo, o modelo de
caixa de areia acaba sendo o responsável por criar diversas novas técnicas e equipamentos
mais ecazes e precisos.
Um dos métodos de mapeamento geológico é a sísmica de reexão, que é muito
utilizada para a exploração de petróleo. Na sísmica de reexão, são utilizados sensores
em embarcações em rios, lagos ou mares, para mapear a superfície do terreno submerso,
onde são lançadas ondas, podendo ser ultrassônicas ou não, para que elas sejam enviadas
através das camadas geológicas e reetidas nas interfaces. As interfaces são as mudanças
entre um tipo de material geológico e outro, ou seja, ao mudar o tipo de material, uma
nova interface geológica surge e reete uma parte da onda que está se propagando pelas
camadas. Ao nal do processo, os estudiosos da área recebem os sinais reetidos por cada
interface e o tempo que leva entre as reexões, o que ajuda a compreender as composições
e as as distâncias entre as camadas. A Figura 1 representa um tipo de sísmica de reexão.
O levantamento sísmico tridimensional (3-D) teve seu desenvolvimento baseado em
fornecer compreensão sobre a estrutura 3-D completa da geologia, provando no processo,
que métodos bidimensionais (2-D) eram inadequados para descrever a geologia 3-D. Duas
grandes vantagens da modelagem física sísmica em relação aos levantamentos de campo
são: que a geologia é conhecida no sistema de caixa de areia, então é possível medir e
compreender diretamente os efeitos das propagações das ondas; e esse tipo de teste é
menos custoso. Para realizar a medição no sistema de caixa de areia é necessário acoplar
um aparato tecnológico acima dele, usando os sensores ultrassônicos para registrar dados
sísmicos 2-D e 3-D, que são controlados por um microcontrolador responsável por controlar
as coordenadas x, y e z do sistema.

2013).
O projeto é baseado em uma aplicação dos conceitos de sistema de caixa de areia e
imageamento sísmico, tendo como principal importância criar uma forma de sensoriamento
em conjunto com a interface gráca que possa demonstrar de forma concreta, como as
camadas da superfície terrestre se comportam em determinadas situações. A interface
gráca foi desenvolvida para ser integrada ao sistema de imageamento sísmico do sistema
de caixa de areia, onde dois (ou mais) sensores ultrassônicos são inseridos em dois eixos
mecânicos, controlados por motores, que movimentam os sensores por todo o plano (x,
y) do sistema, fazendo uma varredura completa e mapeando toda a área. A cada nova
aquisição dos dados, esse valor é então enviado à interface que plota os pontos em sua
área gráca e conecta-os através de retas ou curvas suaves, buscando se assemelhar o mais
elmente possível ao que está sendo mapeado no sistema de caixa de areia.
1.1
Contextualização
Este trabalho foi desenvolvido no Laboratório de Avaliação e Medição em Petróleo
(LAMP) em parceria com os departamentos de geologia e geofísica da Universidade Federal

1.2
Objetivos
Os objetivos do trabalho são: criar um protótipo do sistema de medição de batimetria
aplicado ao sistema de caixa de areia com a utilização de um ESP32 e um sensor HC-SR04,
realizar a batimetria com o protótipo, criar uma interface para imageamento sísmico (onde
o usuário possa interagir com ela de forma simples e rápida), que representa gracamente
a batimetria do sistema, tanto em grácos 2-D quanto em grácos 3-D, e salvar os dados
da batimetria em arquivos de texto e de imagem.
Neste capítulo, são apresentados os conhecimentos necessários para desenvolver o
presente trabalho, expondo a base teórica utilizada.
2.1
Descrição do Sistema para Medição de Interfaces Geológicas
Um dos equipamentos criados a partir do sistema de caixa de areia é um equipamento
de aferição, medição, monitoramento e visualização de interfaces em geologia estrutural. A
aplicação consiste em uma estrutura acima do sistema de caixa de areia conectada por
eixos nas direções x e y, que são movimentados por motores, onde há sensores ultrassônicos
responsáveis pelo mapeamento, nos quais há um emissor responsável por emitir ondas com
determinadas frequŸncias, em cada posição do sistema, e alguns receptores responsáveis por
receber as ondas reetidas nas interfaces da estrutura geológica e armazená-las de forma a
serem estudados os dados e representados por meio de grácos como estão dispostas estas
interfaces.
O diagrama de blocos do sistema é apresentado na Figura 4. Inicialmente, o sistema
funciona através da interface de comando, enviando um comando ao microcontrolador,
ligado ao computador, que, por sua vez, comanda o driver de potŸncia, responsável por
movimentar os motores que modicam as posições x e y do conjunto de sensores. A partir
deste comando, o sistema inicia o deslocamento nos eixos, podendo se mover ponto a ponto,
da coordenada (0,0) do plano x, y, ou iniciando de um ponto especíco, predeterminado
pelo usuário. Ou seja, a partir de comandos na interface, é possível escolher o ponto inicial
de amostragem dos sensores, assim como retornar a um ponto já passado pelo sistema e
refazer a amostragem, caso se deseje observar se houve erros.
Para cada ponto (x, y) que o sistema atua, o microcontrolador envia, uma única
vez por amostragem, um comando ao osciloscópio, que, através do Gerador de onda
presente nele, irá emitir um sinal analógico, que será responsável por iniciar o processo de
transmissão da onda ultrassônica no transmissor (Sensor Tx). Como o sinal que os sensores
ultrassônicos emitem é de baixa amplitude, é necessário inserir um circuito amplicador
(Circuito Amplicador Transmissor) entre o Gerador de onda e o Transmissor, para que o
sinal seja amplicado. Esse sinal transmitido é então enviado em direção à caixa de areia,
naquele dado ponto onde se deseja fazer a aferição das interfaces, e é reetido nas interfaces
daquele ponto, onde cada material diferente será responsável por reetir determinadas
cada camada entre as interfaces e quanto de distância há entre cada interface. Esses sinais
que são reetidos e recebidos pelos receptores são captados por 4 canais do osciloscópio,
cada canal conectado a um único receptor. Entre os canais do osciloscópio e os receptores
também há um circuito amplicador (Circuito Amplicador Receptor) para que seja
possível a visualização dos sinais de saída dos receptores.
Após a amplicação, cada sinal recebido pelos receptores é visualizado no osciloscó-
pio e, em seguida, utilizando-se do Software OpenChoice Desktop, da Tektronix, fabricante
do osciloscópio, que permite visualizar os dados do osciloscópio em formato de onda, esses
sinais são armazenados no computador, em formato de onda ou também em informações
em formato de tabela do Excel. Desta forma, ao nal de todo o mapeamento da caixa de
areia, pode-se agrupar os sinais resultantes das interfaces e entender a partir deles como
as interfaces se comportam em toda a extensão da caixa de areia. Desta forma, facilitando
o processo de análise das interfaces das camadas geológicas.
O projeto do sistema de imageamento sísmico através de um sistema de caixa de
posição dos sensores no eixo (x, y) acima do sistema e são realizadas as medições, onde os
valores de posição x e y e de distância são enviados para o computador pela porta serial.
Esses valores então são processados e enviados para a interface.
2.2
Descrição do Sistema para Batimetria
Uma outra aplicação possível de um Sistema de Caixa de Areia com sensoriamento
através de sensores ultrassônicos é a de batimetria, que consiste em medir e mapear
as profundidades de corpos d’água. Neste caso, os sensores são utilizados apenas na
primeira interface das camadas geológicas; obtendo os valores de distância através dos
sinais rebatidos na camada supercial, e, guardados tais valores, são plotados grácos que
se aproximam do relevo presente naquele corpo d’água.
após isso, serem implementados os valores em (x, y, z).
2.3
Microcontrolador
Um microcontrolador é um dispositivo compacto, integrando em um único chip
funcionalidades de um computador, o que inclui Unidade Central de Processamento (CPU),
memória e periféricos de entrada e saída (I/O). Ou seja, é, basicamente, um sistema
completo de computação em miniatura. São projetados com o intuito de executar tarefas
especícas, sendo utilizados em uma gama de sistemas embarcados(VAHID; GIVARGIS,
2002).
Um microcontrolador possui uma CPU (um microprocessador) além de uma quan-
tidade xa de RAM, ROM, portas de I/O e temporizador, todos em um único chip. Em
outras palavras, o processador, RAM, ROM, portas de I/O e temporizador estão todos
integrados em um único chip; portanto, o projetista não pode adicionar memória externa,
I/O ou temporizador a ele. A quantidade xa de ROM, RAM e o número de portas de
I/O no chip tornam os microcontroladores ideais para muitas aplicações nas quais o custo
e o espaço são críticos(MAZIDI; MAZIDI; CAUSEY, 2008).
uma troca, resultando em uma diminuição da velocidade dos processadores voltados para
computadores e no ganho de memória e periféricos integrados(HOROWITZ; HILL, 2015).
2.4
Sensores Analógicos
Um sensor analógico é um tipo de sensor capaz de medir uma quantidade física, ou
um fenômeno, e, a partir disso, fornecer um sinal de saída diretamente proporcional ao que
está sendo medido. Ou seja, é um sensor que produz um sinal de saída contínuo e analógico,
representando o valor do parâmetro a ser medido com exatidão e sem discretização ou
conversão digital.
Esse tipo de sensor fornece um sinal de saída contínuo e que varia dependendo
das mudanças do parâmetro medido. Produzem como sinal de saída valores de tensão
ou corrente, sendo esse sinal correspondente à magnitude do parâmetro. Alguns fatores
que podem inuenciar negativamente quando se deseja utilizar sensores analógicos são
ruídos e condições ambientais, que podem ocasionar em medições não tão precisas e exatas
quanto se espera. Em cenários onde se espera ter um monitoramento contínuo e em tempo
real das grandezas físicas, sensores analógicos são a melhor opção, dado que a sua saída
representa diretamente a quantidade física medida (TECHNOLOGIES, 2024).
Algumas das aplicações de sensores analógicos são:
1. Sensores de luz: dentre as diversas possibilidades de aplicações, eles podem ser
utilizados em câmeras digitais, controle de iluminação, automação residencial e
industrial, e sistemas de segurança.
2. Sensores de temperatura: são normalmente utilizados quando se quer controlar a
temperatura de um ambiente, podendo ser citados como exemplos: sistemas de
ventilação e ar condicionado (HVAC), eletrodomésticos, sistemas de aquecimento,
controle de processos industriais, entre outros.
3. Sensores de umidade: além de outras aplicações são comumente utilizados em aplica-
ções agrícolas, como no monitoramento da umidade do solo, para melhor controle de
plantações.

2.7
Transdutor
O Transdutor é um dispositivo que converte uma forma de energia em outra. Em
termos de instrumentação eletrônica, eles são aqueles dispositivos que convertem uma
grandeza física em sinal elétrico proporcional, ou o contrário, sendo de vital importância
para o monitoramento, medição e controle de variáveis físicas dentro desses sistemas. Esse
tipo de dispositivo atua como uma interface entre o mundo físico e sistemas eletrônicos,
transformando grandezas mensuráveis (temperatura, pressão, luminosidade, etc) em sinais
elétricos (tensão, corrente e resistŸncia) a serem processados pela instrumentação.
Um transdutor dito como passivo é caracterizado por seu sinal de entrada fornecer
inteiramente a energia de saída. Um exemplo deste tipo de transdutor é um sensor
ultrassônico, pois ele utiliza o efeito piezoelétrico para gerar tensões proporcionais à tensão
mecânica produzida pelas ondas captadas pelo cristal piezoelétrico presente dentro dele.
Transdutores são ditos ativos quando tŸm uma fonte externa de energia responsável por
fornecer a maior parte da potŸncia de saída. Um exemplo é um sensor de temperatura que
utiliza um termistor, porque existe uma tensão elétrica externa constantemente conduzindo
a corrente através do termistor à medida que a resistŸncia varia em resposta às mudanças
de temperatura.
Normalmente, os transdutores são diferenciados por serem utilizados na entrada ou
na saída do sistema, recebendo nomenclaturas distintas: os transdutores de entrada são
chamados de sensores, enquanto os de saída são chamados de atuadores. O primeiro mede e
detecta grandezas físicas e proporciona, a partir delas, um sinal proporcional elétrico para
o sistema e o último faz o caminho oposto, recebendo um sinal elétrico e o transformando
em algum tipo de energia: mecânica, luminosa, térmica. É necessário conhecer a fundo os
limites do sistema antes de escolher quais sensores/atuadores que se deseja utilizar, sendo
algumas das considerações a se fazer: sensibilidade, precisão, linearidade, conabilidade,
etc.

Comunicação de Dados
Esse é um termo utilizado amplamente para denir transmissões de dados de um
ponto a outro. Ao buscar garantir uma comunicação de dados coerentes entre dois ou mais

O padrão mais comum é a comunicação assíncrona, onde é denido que cada carac-
tere de dado a ser transmitido seja iniciado com um bit de ’start’ e seguido de um ou mais
bits ’stop’. Entre os caracteres, existe uma marcação mantida continuamente e, como cada
O UART (Universal Asynchronous Receiver/Transmitter) é um dos mais comuns
dispositivos de comunicação de dados, convencionalmente dependendo de dois registradores
de deslocamento serial separados, cada um tendo sua própria porta serial e seu clock,
para receber e transmitir os dados de um modem ou periférico como resposta a sinais
de controle do sistema de processamento de dados associado. Com essa arquitetura, é
possível enviar e receber dados de forma simultânea em diferentes taxas de dados através
da UART.
Para ocorrer essa transmissão de dados do sistema de processamento de dados
para um modem ou um periférico, a UART pode solicitar uma transferŸncia paralela de
dados para o registrador de retenção do transmissor; normalmente, essa transferŸncia é
um caractere de 8 bits ou byte colocado no barramento de dados do sistema. Após isso,

registrador de retenção está vazio, a UART envia um sinal para a CPU, indicando que
está apta a receber dados. Esses dados são transferidos a partir do momento em que o

a UART (MICHAEL, 1992).
Mesmo que a transmissão serial real ainda não esteja completa, é possível que
a UART indique ao sistema de processamento de dados que pode ser carregado um
novo caractere no registrador de retenção. Isso ocorre porque o registrador de retenção
é considerado "vazio"ao ocorrer a transferŸncia paralela de dados. Quando esses novos
dados são carregados no registrador de retenção, os dados permanecem nele até que o
deslocamento serial dos dados anteriores seja nalizado. Só então a transferŸncia dos novos
dados para o registrador de deslocamento é permitida.

1. Aguardar a conclusão da transferŸncia serial do registrador de deslocamento para
poder continuar a enviar dados.
2. Executar outras tarefas, posteriormente respondendo a várias interrupções geradas
pela UART para concluir a transmissão.

Interface de Comunicação
Uma interface de comunicação permite a troca de informações entre sistemas
ou dispositivos distintos e/ou entre humanos e máquinas, atuando na facilitação da
comunicação, garantindo a transmissão e compreensão adequadas das mensagens.Uma
interface é denida como um espaço de comunicação que possibilita a interação entre o ser
humano e o computador, desta forma, elas argumentam que uma interface é mais do que
uma superfície física, sendo um espaço semiótico onde ocorrem as trocas de signicados
entre o usuário e o sistema (OLIVEIRA; BARANAUSKAS, 2020).
Interfaces são todos os aparatos materiais que permitem a interação entre o universo
da informação digital e o mundo ordinário. Ou seja, uma interface, necessariamente, é
material e media as comunicações que ocorrem entre o ser humano e sistemas digitais
(LEVY, 1999). De toda forma, é indiscutível a importância das interfaces de comunicação
atualmente, sendo determinantes para a apresentação e recebimento de informações, tendo
inuŸncia direta na usabilidade e experiŸncia do usuário. É por este motivo que, para
uma interface ser bem aceita, ela necessita ser de fácil compreensão e navegação. Sistemas
interativos ecazes utilizam essa ideia para facilitar a troca de informações e a realização
de tarefas com maior eciŸncia e de forma mais intuitiva, conectando assim diferentes
mundos, seja uma conexão entre humanos e máquinas ou entre diferentes sistemas.
O termo imageamento sísmico abrange o conjunto de técnicas utilizadas para
investigar e visualizar características do subsolo com alta precisão, como composição,
estrutura e dinâmica, sendo essas técnicas comuns em aplicações como exploração mineral,
monitoramento de riscos, mapeamento geológico e planejamento de obras de infraestrutura.
Essas técnicas envolvem coleta de dados, análise de imagens de satélites, pers sísmicos e
entre outros. As interpretações são responsáveis por ajudar na identicação de padrões de
formação de rochas, características estruturais como falhas e dobras, bem como identicam
propriedades físicas e químicas dos materiais (KEAREY; AL., 2002).
Dentre os métodos de imageamento sísmico podem ser citados: o sensoriamento
remoto, onde são utilizadas imagens de satélite e aerofotogrametria para análise em larga
escala, usando dados multiespectrais para identicar variações litológicas e padrões estru-

e magnéticos que identicam anomalias indicando corpos geológicos enterrados, tais quais
depósitos minerais e intrusões ígneas; e sísmica de reexão, empregado na investigação em
detalhes da subsuperfície, nos quais são geradas ondas sísmicas que são captadas após a
sua reexão em diferentes camadas do solo, gerando assim imagens detalhadas daquela
região estudada.
Sendo assim, o imageamento sísmico é de vital importância em algumas áreas,
como na avaliação de estabilidade de terrenos, onde se pode planejar grandes obras de
infraestrutura apenas com esse tipo de técnica sendo empregada. Também podem ser
empregadas técnicas de imageamento para prever desastres envolvendo deslizamentos de
terra, inundações e terremotos, tornando mais fácil de evacuar locais sob perigo e salvar
vidas.

Distância = TH ∗V S
(3.1)
Em que TH é o tempo em que o pino ECHO está em HIGH e VS é a velocidade do

O módulo FTDI é um conversor USB TTL que utiliza o chip FT232R e possui a

RX e DTR, os quais tŸm uma conguração própria.
1. Pino GND: é o pino de terra ou referŸncia de potencial, responsável por conectar o

3. Pino VCC: é o pino responsável pela alimentação do circuito, podendo ser conectado

4. Pino TX: é o pino "Transmitter"ou transmissor, responsável por enviar os dados

comunicação serial seja estabelecida;
5. Pino RX: é o pino "Receiver"ou receptor, responsável por receber os dados de
outro dispositivo e deve ser conectado ao pino TX de outro dispositivo para que a
comunicação serial seja estabelecida;
6. Pino DTR: é o pino "Data Terminal Ready"ou "terminal de dados pronto", que é
responsável por indicar que o dispositivo terminal, o computador ou microcontrolador,
está pronto para iniciar a comunicação. É um sinal de controle utilizado para informar
ao dispositivo conectado que está ativo e pronto para enviar ou receber dados.

salva os dados em formato .csv, criando uma janela para que o usuário escolha onde deseja
salvar o arquivo e com qual nome deseja salvá-lo. Essa função também confere qual o botão
foi escolhido na interface, se distância ou velocidade, e envia o valor do dado selecionado
junto do valor das posições do eixo x e y. Em suma, essa função salva um arquivo .csv com
esses valores dispostos em trŸs colunas separadas, cada coluna responsável por um dos trŸs
dados recebidos. A segunda função salva o arquivo como imagem, também abrindo uma
janela para o usuário escolher onde e com qual nome deseja salvar seu arquivo, entretanto,
ela captura uma imagem da janela de plotagem da interface e a salva, mostrando quais os
valores dos dados e como se comporta as curvas que conectam os dados plotados.
4.8
Conexão entre Interface e Circuito de Aquisição
A conexão entre a interface de imageamento sísmico e o circuito de aquisição é
extremamente necessária porque ambos precisam utilizar a porta serial do computador;
entretanto, quando um deles a está utilizando, o outro ca impossibilitado de utilizá-la
também, ocasionando problemas de comunicação, onde o dado não pode ser entregue do
microcontrolador para a interface.
Portanto, para ser possível a realização das medições dos valores recebidos pelo
sensor ultrassônico e também conseguir mostrá-los de forma correta no display da interface
perceber que o circuito de aquisição é conectado diretamente ao microcontrolador (no caso
do projeto, é utilizado o ESP32) que se conecta ao computador pela porta serial e por
meio da comunicação Tx-Rx com o módulo FTDI. O módulo, por sua vez, além dessa
conexão com o microcontrolador, também se conecta ao computador através de outra
porta serial. O computador envia os dados da comunicação serial com o microcontrolador
para a Arduino IDE e os dados da comunicação serial com o módulo FTDI para a interface
gráca.
Esse capítulo descreve os resultados obtidos no projeto em questão, representando
quais foram os resultados e descrevendo-os.
5.1
Interface para aquisição 2-D
Como planejado no início, a interface para aquisição de dados no formato 2-D está
completamente funcional e contém todas as funções necessárias: o menu inicial com as
funções de iniciar, parar e reiniciar a plotagem dos dados; os botões de conguração para
salvar como texto e imagem; os botões responsáveis por modicar qual o parâmetro a ser
plotado, se distância ou velocidade; e a janela gráca onde o gráco é plotado na interface.
A Figura 28 representa a interface de imageamento geológico para aquisição 2-D.
As Figuras 29 e 30 representam aquisições realizadas para velocidade e distância,
respectivamente. O eixo x do gráco presente na interface gráca representa a posição x
do eixo x,y do sistema de caixa de areia, enquanto o eixo y do gráco representa distância
ou velocidade, dependendo do tipo de aquisição que o usuário escolher. Nas aquisições
2-D dos dados de velocidade e de distância são plotados os pontos que são recebidos pelos
sensores ultrassônicos e cada ponto é conectado ao próximo através de uma linha reta ou
uma curva suave. Caso os pontos estejam na mesma distância ou velocidade, o gráco
plota uma linha reta entre os dois pontos, e se as distâncias ou velocidades forem distintas,
o gráco plota uma curva suave entre os pontos, buscando se aproximar ao máximo do
que ocorre na aquisição real.

Dados salvos em formato de texto e imagem para aquisição
2-D
Como foi dito anteriormente, uma das funções necessárias na interface desejada era
a de salvar os dados, portanto, para isso foram criadas duas funções na interface: uma
salva arquivos em formato .csv e outra em formato de imagem. As Figuras 31, 32, 33 e
34 representam, respectivamente, dados salvos no formato .csv para valores de distância,
dados salvos no formato .csv para valores de velocidade, dados salvos no formato de
imagem para valores de distância e dados salvos no formato de imagem para valores de
velocidade.
A interface de imageamento geológico para aquisição 3-D tem as mesmas funções
da interface 2-D, porém a janela gráca é congurada para ter 3 eixos e plotar um gráco
3-D, como demonstra a Figura 35. A diferença para a aquisição 3-D é que o gráco agora
plota mais de uma linha, plotando o eixo x como sendo a posição x do sistema de caixa
de areia, o eixo y como sendo a posição y do sistema de caixa de areia e o eixo z como
distância ou velocidade, dependendo do botão selecionado na interface. Assim como para
a aquisição 2-D, são plotados os pontos recebidos pela serial, com os valores de posição x,
y e distância ou velocidade, no plano x, y, z da interface. Os pontos são conectados por
linhas retas ou curvas, dependendo do valor do eixo z de cada ponto; se o eixo z tiver o
mesmo valor, a interface plota uma linha reta conectando os pontos; caso contrário, ela
plota uma curva suave entre os pontos. Uma das ideias a seguir é plotar grácos com
planos que conectam os pontos, além das linhas que os conectam.
Assim como na interface 2-D, a interface 3-D também contém as funções de salvar
arquivos de texto e de imagem. As Figuras 38 e 39 representam, respectivamente, dados
salvos no formato de imagem para valores de velocidade e dados salvos no formato de
imagem para valores de distância. Infelizmente, a função para salvar em formato .csv não
foi implementada por completo, apenas sendo possível selecionar o botão e abrindo o menu
utuante, entretanto, ao clicar em salvar a interface é fechada, pois ocorre um erro.
Os objetivos estabelecidos neste estudo foram atingidos, uma vez que a interface
para imageamento sísmico foi construída e se provou funcional, sendo possível plotar os
dados adquiridos através dos sensores ultrassônicos nela de forma correta, além de fornecer
uma visualização do imageamento sísmico de um sistema de caixa de areia conhecido
como batimetria. E também se provou que é possível conectar a interface não somente
ao protótipo, como também ao sistema de imageamento sísmico completo, tendo sido
realizada a medição 2-D com essa integração, com os pontos e linhas que os conectam
sendo representadas de forma aproximada.
Outro ponto importante a ser lembrado é que os dados também podem ser salvos
através da própria interface, como era esperado ao iniciar o projeto. Também é relevante
considerar que o projeto de uma interface de imageamento geológico é de fundamental
importância quando se fala em medições deste tipo, pois auxilia prossionais da área de
geologia e geofísica nas medições de sistemas de caixa de areia, além de proporcionar a
professores das áreas em questão um modo de mostrar, praticamente, como esse tipo de
medição é feita no campo, sem que seja necessário sair do ambiente universitário.
