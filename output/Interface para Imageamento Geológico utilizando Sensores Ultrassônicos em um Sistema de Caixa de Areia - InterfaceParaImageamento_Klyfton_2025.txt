Tolkien, Os Filhos de Húrin ()) RESUMO A modelagem física sísmica é uma área da geofísica responsável por modelar como se comporta o subsolo de determinados locais e uma das formas de se realizar essa modelagem envolve a utilização de sensores ultrassônicos em um sistema de caixa de areia. Uma utilização desse sistema de caixa de areia é em batimetria, que é o imageamento geológico da superfície daquele sistema. O objetivo deste projeto é criar uma interface capaz de receber os dados de um sistema com sensores ultrassônicos e realizar esta modelagem em formato de gráco, para que seja possível visualizar de forma completa o imageamento geológico. Para isso, foi utilizado um protótipo do sistema de medição de caixa de areia, com um ESPe um sensor ultrassônico HC-SR, e criada uma interface para modelagem -D e -D utilizando-se o Qt Designer e o VSCode, onde os dados são plotados na interface de aquisição e podem ser salvos em formatos de texto .csv e de imagem, para estudos posteriores. Palavras-chaves: Imageamento Sísmico; Sensores Ultrassônicos; Interface de Aquisição; Sistema de Caixa de Areia. ABSTRACT Seismic physical modeling is an area of geophysics responsible for modeling how the subsurface behaves in specic locations, and one of the ways to perform this modeling involves the use of ultrasonic sensors in a sandbox system. One application of this sandbox system is bathymetry, which is the geological imaging of the surface of that system. The goal of this project is to create an interface capable of receiving data from a system with ultrasonic sensors and performing this modeling in the form of a graph, so that the geological imaging can be fully visualized. To achieve this, a prototype of the sandbox measurement system was used, with an ESPand an HC-SRultrasonic sensor, and an interface for -D and -D modeling was created using Qt Designer and VSCode, where the data is plotted in the acquisition interface and can be saved in text .csv and image formats for later studies. INTRODUÇÃO O sistema de caixa de areia é um modelo de imaginação sísmica de fundamental importância quando se planeja fazer estudos, ou experimentos, geológicos e geofísicos. Eles permitem a observação da evolução das estruturas, tornando possível uma interpretação dos sistemas estruturais mais eciente (KEAREY; AL., ). É através deste sistema que são possíveis simulações de processos geológicos complexos, como formações de falhas, dobras e zonas de cisalhamento, em ambientes controlados, onde pode-se estudá-los em uma escala de tempo menor do que a que ocorre na vida real. Por meio deste tipo de sistema é que são validados os modelos teóricos, comparando dados obtidos em laboratório com os obtidos em campo, ajustando e melhorando a partir disso as técnicas de interpretação. Nesse tipo de sistema, quem controla as condições é o construtor do modelo, podendo comparar os resultados da modelagem com exemplos geológicos naturais. Além disso, é um equipamento perfeito se tratando de educação e treinamento, sendo útil para ensinar estudantes e pesquisadores na visualização e entendimento das forças tectônicas atuantes na formação de estruturas geológicas. Por ser amplamente utilizado para desenvolver e testar novos métodos sísmicos e equipamentos em menores escalas e em ambientes mais controlados, antes de serem levados a campo, o modelo de caixa de areia acaba sendo o responsável por criar diversas novas técnicas e equipamentos mais ecazes e precisos. Um dos métodos de mapeamento geológico é a sísmica de reexão, que é muito utilizada para a exploração de petróleo. Na sísmica de reexão, são utilizados sensores em embarcações em rios, lagos ou mares, para mapear a superfície do terreno submerso, onde são lançadas ondas, podendo ser ultrassônicas ou não, para que elas sejam enviadas através das camadas geológicas e reetidas nas interfaces. As interfaces são as mudanças entre um tipo de material geológico e outro, ou seja, ao mudar o tipo de material, uma nova interface geológica surge e reete uma parte da onda que está se propagando pelas camadas. Ao nal do processo, os estudiosos da área recebem os sinais reetidos por cada interface e o tempo que leva entre as reexões, o que ajuda a compreender as composições e as as distâncias entre as camadas. O levantamento sísmico tridimensional (-D) teve seu desenvolvimento baseado em fornecer compreensão sobre a estrutura -D completa da geologia, provando no processo, que métodos bidimensionais (-D) eram inadequados para descrever a geologia -D. Duas grandes vantagens da modelagem física sísmica em relação aos levantamentos de campo são: que a geologia é conhecida no sistema de caixa de areia, então é possível medir e compreender diretamente os efeitos das propagações das ondas; e esse tipo de teste é menos custoso. Para realizar a medição no sistema de caixa de areia é necessário acoplar um aparato tecnológico acima dele, usando os sensores ultrassônicos para registrar dados sísmicos -D e -D, que são controlados por um microcontrolador responsável por controlar as coordenadas x, y e z do sistema. O conceito da sísmica de reexão utilizada no sistema de caixa de areia também pode ser aplicado para a realização de batimetria, que é a medição da profundidade dos oceanos, lagos e rios. Na medição de batimetria é necessário a utilização de aparelhos responsáveis por medir os dados de posição e profundidade. Existem duas metodologias na realização de batimetria, a forma direta e a indireta, a primeira envolve a utilização de equipamentos que fazem leituras físicas, medindo profundidades de forma tradicional Capítulo . Introdução diretamente no local; a segunda envolve sensoriamento remoto ou modelagem matemática na estimativa das profundidades, sem a necessidade de estar sicamente no local para realizar as medições. É nesse conceito que o projeto aqui apresentado será aplicado(LOPES, ). O projeto é baseado em uma aplicação dos conceitos de sistema de caixa de areia e imageamento sísmico, tendo como principal importância criar uma forma de sensoriamento em conjunto com a interface gráca que possa demonstrar de forma concreta, como as camadas da superfície terrestre se comportam em determinadas situações. A interface gráca foi desenvolvida para ser integrada ao sistema de imageamento sísmico do sistema de caixa de areia, onde dois (ou mais) sensores ultrassônicos são inseridos em dois eixos mecânicos, controlados por motores, que movimentam os sensores por todo o plano (x, y) do sistema, fazendo uma varredura completa e mapeando toda a área. A cada nova aquisição dos dados, esse valor é então enviado à interface que plota os pontos em sua área gráca e conecta-os através de retas ou curvas suaves, buscando se assemelhar o mais elmente possível ao que está sendo mapeado no sistema de caixa de areia. .Contextualização Este trabalho foi desenvolvido no Laboratório de Avaliação e Medição em Petróleo (LAMP) em parceria com os departamentos de geologia e geofísica da Universidade Federal do Rio Grande do Norte (UFRN) e com a Petrobrás, buscando desenvolver um sistema de imageamento sísmico para um sistema de caixa de areia. .Objetivos Os objetivos do trabalho são: criar um protótipo do sistema de medição de batimetria aplicado ao sistema de caixa de areia com a utilização de um ESPe um sensor HC-SR, realizar a batimetria com o protótipo, criar uma interface para imageamento sísmico (onde o usuário possa interagir com ela de forma simples e rápida), que representa gracamente a batimetria do sistema, tanto em grácos -D quanto em grácos -D, e salvar os dados da batimetria em arquivos de texto e de imagem. REFERENCIAL TEÓRICO Neste capítulo, são apresentados os conhecimentos necessários para desenvolver o presente trabalho, expondo a base teórica utilizada. .Descrição do Sistema para Medição de Interfaces Geológicas Um dos equipamentos criados a partir do sistema de caixa de areia é um equipamento de aferição, medição, monitoramento e visualização de interfaces em geologia estrutural. A aplicação consiste em uma estrutura acima do sistema de caixa de areia conectada por eixos nas direções x e y, que são movimentados por motores, onde há sensores ultrassônicos responsáveis pelo mapeamento, nos quais há um emissor responsável por emitir ondas com determinadas frequŸncias, em cada posição do sistema, e alguns receptores responsáveis por receber as ondas reetidas nas interfaces da estrutura geológica e armazená-las de forma a serem estudados os dados e representados por meio de grácos como estão dispostas estas interfaces. O diagrama de blocos do sistema é apresentado na Figura . Inicialmente, o sistema funciona através da interface de comando, enviando um comando ao microcontrolador, ligado ao computador, que, por sua vez, comanda o driver de potŸncia, responsável por movimentar os motores que modicam as posições x e y do conjunto de sensores. A partir deste comando, o sistema inicia o deslocamento nos eixos, podendo se mover ponto a ponto, da coordenada (,) do plano x, y, ou iniciando de um ponto especíco, predeterminado pelo usuário. Ou seja, a partir de comandos na interface, é possível escolher o ponto inicial de amostragem dos sensores, assim como retornar a um ponto já passado pelo sistema e refazer a amostragem, caso se deseje observar se houve erros. Para cada ponto (x, y) que o sistema atua, o microcontrolador envia, uma única vez por amostragem, um comando ao osciloscópio, que, através do Gerador de onda presente nele, irá emitir um sinal analógico, que será responsável por iniciar o processo de transmissão da onda ultrassônica no transmissor (Sensor Tx). Como o sinal que os sensores ultrassônicos emitem é de baixa amplitude, é necessário inserir um circuito amplicador (Circuito Amplicador Transmissor) entre o Gerador de onda e o Transmissor, para que o sinal seja amplicado. Esse sinal transmitido é então enviado em direção à caixa de areia, naquele dado ponto onde se deseja fazer a aferição das interfaces, e é reetido nas interfaces daquele ponto, onde cada material diferente será responsável por reetir determinadas faixas de frequŸncia presentes no sinal original. Esse sinal analógico, composto por vários harmônicos, emitido pelo Gerador de onda é chamado de Wavelet. Capítulo . REFERENCIAL TEÓRICO Assim que a Wavelet é reetida em cada uma das interfaces do ponto em que se deseja fazer aferição, os sinais são recebidos por cada um dos receptores (Sensores Rx) e, dependendo do tempo que cada faixa de frequŸncia retorna aos receptores, é possível saber quantas interfaces aquele ponto possui, quais os tipos de materiais presentes em cada camada entre as interfaces e quanto de distância há entre cada interface. Esses sinais que são reetidos e recebidos pelos receptores são captados por canais do osciloscópio, cada canal conectado a um único receptor. Entre os canais do osciloscópio e os receptores também há um circuito amplicador (Circuito Amplicador Receptor) para que seja possível a visualização dos sinais de saída dos receptores. Após a amplicação, cada sinal recebido pelos receptores é visualizado no osciloscó- pio e, em seguida, utilizando-se do Software OpenChoice Desktop, da Tektronix, fabricante do osciloscópio, que permite visualizar os dados do osciloscópio em formato de onda, esses sinais são armazenados no computador, em formato de onda ou também em informações em formato de tabela do Excel. Desta forma, ao nal de todo o mapeamento da caixa de areia, pode-se agrupar os sinais resultantes das interfaces e entender a partir deles como as interfaces se comportam em toda a extensão da caixa de areia. Desta forma, facilitando o processo de análise das interfaces das camadas geológicas. Figura – Diagrama do sistema. Fonte: Elaborado pelo autor. REFERENCIAL TEÓRICO .Comunicação UART O UART (Universal Asynchronous Receiver/Transmitter) é um dos mais comuns dispositivos de comunicação de dados, convencionalmente dependendo de dois registradores de deslocamento serial separados, cada um tendo sua própria porta serial e seu clock, para receber e transmitir os dados de um modem ou periférico como resposta a sinais de controle do sistema de processamento de dados associado. Com essa arquitetura, é possível enviar e receber dados de forma simultânea em diferentes taxas de dados através da UART. Para ocorrer essa transmissão de dados do sistema de processamento de dados para um modem ou um periférico, a UART pode solicitar uma transferŸncia paralela de dados para o registrador de retenção do transmissor; normalmente, essa transferŸncia é um caractere de bits ou byte colocado no barramento de dados do sistema. Após isso, o registrador transfere esses dados para um registrador de deslocamento do transmissor, responsável por transmitir serialmente para o dispositivo periférico cada bit. Quando o registrador de retenção está vazio, a UART envia um sinal para a CPU, indicando que está apta a receber dados. Esses dados são transferidos a partir do momento em que o sistema envia um pulso adequado no data strobe (entrada de sincronização de dados) para a UART (MICHAEL, ). Mesmo que a transmissão serial real ainda não esteja completa, é possível que a UART indique ao sistema de processamento de dados que pode ser carregado um novo caractere no registrador de retenção. Isso ocorre porque o registrador de retenção é considerado "vazio"ao ocorrer a transferŸncia paralela de dados. Quando esses novos dados são carregados no registrador de retenção, os dados permanecem nele até que o deslocamento serial dos dados anteriores seja nalizado. Só então a transferŸncia dos novos dados para o registrador de deslocamento é permitida. Executar outras tarefas, posteriormente respondendo a várias interrupções geradas pela UART para concluir a transmissão. Ou seja, de forma inicial, o circuito de aquisição apenas poderia realizar uma aquisição através do pressionar do botão, tornando a aquisição obrigatoriamente manual nesse ponto. Em seguida, o circuito foi modicado para que, ao pressionar o botão, a aquisição seguisse de forma contínua, fazendo uma varredura completa de todos os pontos do eixo x,y do sistema de caixa de areia. .Criação da Interface de Aquisição de dados A criação da interface de aquisição de dados se baseou em duas partes: primeiro foi utilizado o Qt Designer para criar a interface de forma gráca e segundo foi utilizado o VSCode para criar o algoritmo responsável por conectar os Widgets, criados através do Qt Designer, com as suas respectivas funções na aplicação da interface. Portanto, para ser possível a realização das medições dos valores recebidos pelo sensor ultrassônico e também conseguir mostrá-los de forma correta no display da interface de imageamento sísmico, foi necessário realizar a comunicação entre eles, sendo utilizado o conversor FTDI USB Serial TTL e RSSerial - FTR nesta comunicação, onde ele foi conectado ao ESP, com os pinos GND de ambos conectados ao mesmo ponto; o pino TX, que é o transmissor do conversor FTDI, conectado ao pino RX do ESPe o pino RX, que é o receptor do conversor FTDI, conectado ao TX do ESP. Tanto o microcontrolador quanto o conversor FTDI utilizavam uma porta USB, ou porta serial, distinta do computador, conectados às portas COMe COM, respectivamente, para a realização da comunicação. Apenas após essa conexão é que tornou-se possível haver comunicação entre microcontrolador e interface, sendo então possível a visualização dos dados na interface.
