Resumo Este trabalho teve como objetivo desenvolver uma solução técnica, utilizando análise baseada em linguagem natural, mais especiﬁcamente de Modelos de Linguagem de Larga escala (Large Language model - LLMs), para facilitar o acesso aos dados legislativos da Câmara dos Deputados, com uma interface que proporciona uma experiência agradável, de fácil acesso que permite a execução de consultas de informações para análise. Para isso, foram coletados os dados em formato JSON disponibilizados pela Câmara dos De- putados e transformados em um banco de dados relacional MySQL. O sistema faz o uso de uma LLM, integrada ao framework LangChain, que gera con- sultas SQL a partir de uma entrada de texto feita em linguagem natural, essa consulta é executada na base de dados e os resultados obtidos são apresentados em uma interface gráﬁca desenvolvida utilizando o framework Streamlit, que fornece ao usuário um campo especíﬁco para realizar perguntas e uma exibição das respostas obtidas. Palavras-chave: Dados legislativos, inteligência artiﬁcial, linguagem natural, Lang- Chain, Streamlit. Abstract This work aimed to develop a technical solution using natural language analysis, spe- ciﬁcally Large Language Models (LLMs), to facilitate access to legislative data from the Brazilian Chamber of Deputies. The solution includes an interface designed to provide an intuitive and user-friendly experience, allowing users to execute queries for information analysis. To achieve this, JSON-format data provided by the Chamber of Deputies was collected and transformed into a relational MySQL database. The system leverages an LLM integrated with the LangChain framework, which ge- nerates SQL queries based on natural language input. These queries are executed in the database, and the results are displayed in a graphical interface developed using the Stream- lit framework. This interface offers users a dedicated input ﬁeld for queries and presents the corresponding responses. Keywords: Legislative data, artiﬁcial intelligence, natural language, LangChain, Stre- amlit. Tal fenômeno, vem possibilitando a criação de projetos Web que torna mais acessível o acesso aos dados à população. Segundo o portal Dados Abertos da Câmara dos Deputados(CÂMARA DOS DEPU- TADOS, ), em vários países do mundo, as pessoas estão cada vez mais interessadas e exigentes em relação à transparência no trabalho de seus governantes e das instituições e empresas públicas. Em resposta a esse desejo, as instituições onde esse trabalho acontece têm tornado suas informações cada vez mais públicas. No Brasil, por exemplo, a Lei de Acesso à Informação (LAI), de , garante o direito de solicitar e receber informações de órgãos públicos, sem ter que pagar por isso, para qualquer cidadão ou entidade. No Brasil, a Câmara dos Deputados disponibiliza uma vasta quantidade de informa- ções públicas em formato digital. Entretanto, o formato de disponibilização desses dados é bastante técnico, sendo feito por meio de arquivos que não permitem uma fácil visuali- zação como JSON e CSV. Contudo, apesar de serem arquivos de fácil acesso, percebe-se que ao mesmo tempo diﬁcultam uma análise mais qualitativa e prática desses dados, sendo necessário um conhecimento mais aprofundado em ferramentas de manipulação de dados, para manipulá-los e assim realizar consultas e extrair informações que sejam úteis. Neste contexto, é possível notar que existem sistemas Web que resolvem esse pro- blema de acesso às informações, dentre eles, podemos citar: o projeto Akan, que facilita a criação de aplicações que utilizam dados públicos, disponível no Github, entre outros que estão disponíveis na comunidade do site dados abertos. Todavia, essas consultas ainda são limitadas e muito direcionadas ao contexto da aplicação. Paralelamente, os avanços em inteligência artiﬁcial, principalmente com o advento recente dos Modelos de Linguagem de Larga Escala, abrem novas oportunidades para transformar a forma como interagimos com grandes volumes de dados. Ferramentas como o LangChain possibilitam integrar LLMs a bancos de dados, viabilizando a tradução de consultas em linguagem natural para comandos SQL precisos e eﬁcientes(LANGCHAIN, ). Desse modo, o presente trabalho tem como objetivo desenvolver uma solução que integre dados legislativos da Câmara dos Deputados a uma plataforma de inteligência artiﬁcial capaz de realizar consultas e análises de maneira intuitiva e acessível. A pro- posta abrange três etapas principais: (i) extração e transformação de dados legislativos em JSON para um banco de dados relacional MySQL; (ii) utilização de uma LLM utili- zando LangChain, que gera consultas SQL a partir de perguntas feitas por usuários; e (iii) desenvolvimento de uma interface interativa com Streamlit, permitindo que qualquer pes- soa, independentemente do seu conhecimento técnico, possa explorar e analisar os dados legislativos. Assim, ao conectar os domínios de ciência de dados, inteligência artiﬁcial e trans- parência governamental, este trabalho visa facilitar o acesso a informações legislativas e promover o uso consciente dos dados públicos. Os modelos de ER são fundamentais no design de bancos de dados, pois permitem a representação abstrata das estruturas de dados e seus relacionamentos. Essas tarefas incluem percepção, raciocínio, tomada de decisão e até mesmo a comunicação em linguagem natural. A IA é uma das tecnologias mais transformadoras da atualidade e tem sido aplicada em diversos setores, incluindo saúde, ﬁnanças, educação e, mais recentemente, em processos legislativos e análise de dados (Russell & Norvig, ). O Processamento de Linguagem Natural é um subcampo da área de computação e da inteligência artiﬁcial, que tem como foco as interações entre linguagem humana e com- putadores. O PLN engloba tarefas como tradução automática, resumo de texto, análise de sentimentos e, no contexto de sistemas de consulta, a conversão de linguagem natural em comandos que um computador pode executar, como a transformação de uma consulta em linguagem natural em uma query SQL. Conforme Manning e Schütze (), "o proces- samento de linguagem natural permite que os computadores compreendam e respondam adequadamente às solicitações feitas em linguagem humana". Esse processo de tradução de texto livre para uma linguagem de consulta estruturada envolve uma série de tarefas de PLN, como: . Análise Sintática: Identiﬁcar as partes da pergunta e suas relações. Extração de Entidades: Determinar os elementos chave da pergunta, como datas, autores e tipos de proposições. Essas entidades são usadas para construir a consulta SQL. . Contextualização da Pergunta: Através do uso de prompts e modelos de lingua- gem, o sistema é capaz de identiﬁcar o contexto da consulta e gerar uma resposta precisa com base na estrutura do banco de dados. Por exemplo, ele pode discernir que uma "proposição legislativa"está associada a tabelas de "proposições" e "vota- ções". Para o funcionamento do sistema é necessário criar uma cadeia de execuções interli- gadas que o resultado de uma seja a entrada da outra, pois dessa forma é possível criar uma pipeline de execuções que responsabiliza cada etapa em um determinado escopo, permitindo assim executar qualquer ação necessária entre os estados, como ilustrado na ﬁgura .. Fonte: Próprio Autor O funcionamento principal deste sistema envolve a conversão de perguntas em lin- guagem natural, formuladas pelos usuários, em consultas SQL executáveis no banco de dados. A chave para essa transformação é utilizar um modelo de linguagem avançado, que seja capaz de compreender a intenção por trás de uma pergunta e gerar a consulta SQL apropriada. O modelo tem a função de interpretar o texto de entrada, reconhecer a consulta implícita e converter essa consulta para uma linguagem estruturada, como o SQL.
